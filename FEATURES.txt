SlotSwapper - Feature Checklist
================================

CORE FEATURES
=============

✅ User Authentication
  ✅ Sign Up with name, email, password
  ✅ Log In with email and password
  ✅ JWT token generation (7 day expiry)
  ✅ Bearer token authentication on all protected routes
  ✅ Password hashing with bcrypt
  ✅ Input validation (Joi)
  ✅ Error handling for duplicate emails
  ✅ Secure session management

✅ Backend: Calendar & Data Model
  ✅ Database schema (PostgreSQL)
    ✅ users table
    ✅ events table with status enum
    ✅ swap_requests table with status enum
    ✅ Foreign key relationships
    ✅ Timestamps (created_at, updated_at)
    ✅ Indexes for performance
    ✅ Check constraints
  ✅ Event CRUD API endpoints
    ✅ GET /api/events - Get all user events
    ✅ GET /api/events/:id - Get single event
    ✅ POST /api/events - Create event
    ✅ PUT /api/events/:id - Update event
    ✅ DELETE /api/events/:id - Delete event
  ✅ Event status management (BUSY, SWAPPABLE, SWAP_PENDING)
  ✅ Authorization checks (users can only modify their own events)

✅ Backend: Swap Logic
  ✅ GET /api/swappable-slots
    ✅ Returns all SWAPPABLE slots from other users
    ✅ Excludes current user's slots
    ✅ Includes owner information
  ✅ POST /api/swap-request
    ✅ Validates both slots exist
    ✅ Verifies both slots are SWAPPABLE
    ✅ Creates SwapRequest record with PENDING status
    ✅ Updates both slots to SWAP_PENDING
    ✅ Prevents swapping with self
    ✅ Sends real-time notification to target user
  ✅ POST /api/swap-response/:requestId
    ✅ Validates user is target of request
    ✅ Verifies request is PENDING
    ✅ Accept path:
      ✅ Marks request as ACCEPTED
      ✅ Swaps event owners atomically (database transaction)
      ✅ Sets both events to BUSY
      ✅ Sends real-time notification to requester
    ✅ Reject path:
      ✅ Marks request as REJECTED
      ✅ Restores both slots to SWAPPABLE
      ✅ Sends real-time notification to requester
  ✅ GET /api/requests/incoming - View incoming swap requests
  ✅ GET /api/requests/outgoing - View outgoing swap requests

✅ Frontend: UI/UX

  ✅ Notifications/Requests View
